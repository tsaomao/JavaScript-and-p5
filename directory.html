Introduction

Greetings and Welcome! This file is named "directory.html" so that it doesn't override your ability to browse these files and folders - in case I don't keep up with the updates while I create content and examples here.

These branching subdirectories will host the javascript and HTML that I create while learning p5.js and matter.js from The Coding Train.

The root directory for all this is hosted on my website.

Here's the GitHub Repository.

In almost all cases you need to open a web page that will in turn exectute Javascript within the browser window, so all you should need to view these examples is a web browser. Using GitHub, you can also view the source code within your browser.

Directory

  • Setup
    Notes and files on how I set this workflow up.
    • Setup Notes
      How I got Git deployments working.
  • Templates
    For templates provided by packages, or templates I've made to make further coding and writing more convenient.
    • p5-empty-example
      Examples provided here come directly from the p5.js project. The only change I made (for purposes of browsing via HTTP/WebDAV) is to change the filenames from index.html to index1.html.
  • p5 Exp
    My own experiments or inspired by early lessons based on The Coding Train. Note that each index*.html corresponds to a sketch*.js file, which is what gets executed in the index*.html in the browser.
  • Lessons
    I'll come up with some sort of nomenclature in subdirectories or filenames that corresponds in some way to The Coding Train's video lectures and exercises.
    • Algorithmic_Botany
      Corresponds to lessons available in the Algorithmic_Botany (The Coding Train)
      • indexcc14.html - Corresponds to Coding Challenge 14: Fractal Trees - Recursive
    • The Nature of Code - Physics Engines
      Studying using matter.js 2D physics engine AND p5.js to explore Javascript physics engines.
      • index5-17.html
      • index5-17a.html - My further explorations of similar coding.

It's also my intention to comment the JavaScript files so that the code is somehow explicable to the casual browser.
